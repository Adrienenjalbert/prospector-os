## The 5 design decisions that determine adoption

1. **Intelligence vs Automation** — 63% of sellers prioritise AI for
   qualification, deal strategy, solution design. NOT admin
   automation. Build for *effectiveness*, not efficiency.
2. **Personal vs Platform** — Platform AI plateaus at ~1% adoption.
   Per-rep agents that personalise to territory, deals, selling style
   report 3x pipeline growth. The agent must feel like *my* assistant.
3. **Push vs Pull** — Reps don't have time to query during live calls.
   Default interaction = system *tells* the rep something. Push is
   the entry point; pull is the depth layer.
4. **Explainable vs Opaque** — Only 7% feared AI replacing them. Real
   concerns: accuracy, workflow fit, transparency. Every recommendation
   needs a receipt.
5. **Progressive vs Comprehensive** — Layer info in 2–3 tiers. Lead
   with action. Show reasoning on request. Hide methodology unless
   asked.

## The 3 fatal mistakes

1. **Value Requires Effort Before Delivery** — Time to first value
   must be measured in hours, not weeks. Configuration cost = trust
   debt.
2. **Adding Cognitive Load Instead of Removing It** — If the AI
   surfaces 20 accounts but doesn't tell the rep which 3 to call
   first, you've added noise. Every output must end with a smaller
   decision than the rep started with.
3. **Measuring Feature Usage Instead of Habit Formation** — DAU is
   a poor predictor. The best predictor: does the product create a
   *repeatable habit loop*?
